# Competitions Page — Transfer Package

Everything you need to recreate the Competitions feature in another Replit
session, with the **same data and thumbnails**, while keeping your own layout.

## What's in this folder

| File | What it is |
|------|------------|
| `competitions-data.json` | All 3 competitions + 55 games, including thumbnail URLs. The actual data. |
| `CompetitionsPage.tsx` | The competitions grid page (list of competitions). |
| `CompetitionGalleryPage.tsx` | A single competition's app gallery (`/competitions/:id`). |
| `schema-snippet.ts` | The database tables (`competitions`, `games`) to add to `shared/schema.ts`. |
| `backend-snippet.ts` | The Express API routes + storage methods (commented, copy what you need). |
| `import-data.ts` | Script that loads `competitions-data.json` into your database. |

> **Note on thumbnails:** every thumbnail is stored as a URL inside the JSON, so
> nothing extra needs to be copied — importing the data brings the images along.

## Setup steps (in the other session)

### 1. Add the database tables
Open `schema-snippet.ts` and copy the `competitions` table (and the
`competition_id` column on `games`) into your `shared/schema.ts`. Then push the
schema:
```bash
npm run db:push
```

### 2. Add the backend routes
Open `backend-snippet.ts` and copy the routes into your `server/routes.ts` and
the storage methods into your `server/storage.ts`. The endpoints are:
- `GET /api/competitions`
- `GET /api/competitions/:id`
- `GET /api/competitions/:id/games`
- `POST /api/competitions`

### 3. Import the data
Copy this whole `competitions-export/` folder into the other project, then run:
```bash
tsx competitions-export/import-data.ts
```
This loads all competitions and games (with thumbnails) into your database.

### 4. Add the page components
Copy `CompetitionsPage.tsx` and `CompetitionGalleryPage.tsx` into your
`client/src/pages/` and register the routes (wouter, react-router, etc.):
```tsx
<Route path="/competitions" component={CompetitionsPage} />
<Route path="/competitions/:competitionId" component={CompetitionGalleryPage} />
```

## Adapting to YOUR layout

The components only own the **content** of the page (the competition grid and the
gallery grid). They do **not** include any sidebar, top nav, or page chrome —
those come from your own layout wrapper. So you can drop these straight into your
different layout.

Things you'll likely want to tweak for your design:
- **Colors:** the components use classes like `text-neon-orange`, `neon-green`,
  `neon-yellow`, `neon-purple`. Swap these for your own Tailwind theme tokens.
- **Grid columns:** `grid-cols-1 md:grid-cols-2 lg:grid-cols-3` — change to match
  your spacing/density.
- **Header section:** the `<h1>` + "Create Competition" button block at the top is
  optional. Remove or reorder it to fit where your layout puts page titles.
- **Routing import:** these use `react-router-dom` (`useNavigate`, `useParams`).
  If your other session uses `wouter`, swap to `useLocation` / `useRoute`.

That's it — the data, thumbnails, and logic are fully portable; only the styling
classes and routing imports may need adjusting to your layout.
